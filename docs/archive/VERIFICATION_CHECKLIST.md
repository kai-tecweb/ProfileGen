# 改善確認チェックリスト

## 確認が必要な項目

### 1. 記事編集画面で本文が改善されているか
**確認方法**:
1. 既存の記事を編集画面で開く
2. 本文のtextareaを確認
3. ✅ 改行が保持されているか
4. ✅ 読みやすいフォントサイズ（14px）になっているか
5. ✅ 長いテキストが折り返されているか

**問題がある場合**:
- ブラウザのキャッシュをクリア（Ctrl+Shift+R または Cmd+Shift+R）
- フロントエンドのビルドファイルがサーバーに反映されているか確認

### 2. 記事一覧画面で情報が表示されているか
**確認方法**:
1. 「記事管理」画面を開く
2. ✅ タイトルカラムが表示されているか
3. ✅ URLカラムが表示されているか
4. ✅ 本文プレビューカラムが表示されているか
5. ✅ 作成日時カラムが表示されているか

**問題がある場合**:
- ブラウザをリロード（F5）
- フロントエンドのビルドファイルを確認

### 3. HTMLパース時に画像URLが抽出されているか
**確認方法**:
1. 「記事管理」→「新規登録」
2. 「HTMLから本文を自動抽出する」をチェック
3. 画像URLを含むHTMLソースを貼り付け
4. 「保存」をクリック
5. 記事一覧画面でURLカラムに画像URLが表示されるか確認

**問題がある場合**:
- `storage/logs/laravel.log`でエラーログを確認
- HTMLパース処理が動作しているか確認

### 4. 画像URLが保存されているか
**確認方法**:
- データベースで`articles`テーブルの`image_urls`カラムを確認
- または、記事一覧画面のURLカラムで確認

## トラブルシューティング

### フロントエンドの変更が反映されない場合

1. **ビルドファイルを再生成**:
   ```bash
   npm run build
   ```

2. **サーバーにアップロード**:
   ```bash
   # public/buildディレクトリをサーバーにアップロード
   ```

3. **ブラウザのキャッシュをクリア**:
   - Ctrl+Shift+R（Windows/Linux）
   - Cmd+Shift+R（Mac）

### バックエンドの変更が反映されない場合

1. **キャッシュをクリア**:
   ```bash
   php artisan config:clear
   php artisan route:clear
   php artisan view:clear
   ```

2. **サーバーにファイルをアップロード**:
   - `app/Http/Controllers/ArticleController.php`
   - `app/Models/Article.php`

- ✅ 記事登録画面改善済み
- ✅ ビルド成功
